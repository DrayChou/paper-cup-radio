# Paper Cup Radio Testing Guide

## 目标

验证手机输入到桌面总台这条主链路是否稳定。

## 测试前准备

- 电脑和手机处于同一局域网
- 电脑端已打开 Paper Cup Radio
- 手机浏览器可访问首页 `/`

## 基础流程

1. 手机打开首页
2. 给纸杯起一个名字
3. 输入一段文本并发送
4. 电脑端检查：
   - 总台是否收到新播报
   - 是否自动复制到本机剪贴板
   - 历史记录里是否有该条内容

## macOS 打开失败的处理

如果下载后的 `PaperCupRadio.app` 被 macOS 提示“已损毁，无法打开”，通常不是包真的损坏，而是系统拦截了未公证的测试应用。

执行：

```bash
xattr -dr com.apple.quarantine /Applications/PaperCupRadio.app
```

如果 app 不在 `/Applications`，把路径替换成实际位置，再重新打开即可。

## 建议测试项

### 1. 单设备基础发送
- 连续发送 3 到 5 条短文本
- 检查顺序是否正确

### 2. 长文本发送
- 发送多行文本
- 检查换行是否保留

### 3. 随机命名体验
- 多次点击“换个名字”
- 检查命名是否自然、有变化

### 4. 多设备并发
- 两台手机分别命名并发送
- 检查来源设备名是否正确

### 5. 主题切换
- 手机端切换深浅色
- 总台切换深浅色
- 刷新后检查本地保存是否生效

### 6. 竖版窗口
- 将总台窗口调成偏竖版
- 检查底部悬浮分页按钮是否清晰可点

### 7. 剪贴板
- 检查自动复制是否成功
- 手动点击再次复制是否成功
- Windows 下重点检查中文内容是否仍然乱码

### 8. 当前焦点自动粘贴
- 在电脑上先手动点中浏览器或文本框输入位置
- 手机发送一段文本
- 检查文本是否直接进入当前焦点输入框
- 如果自动粘贴失败，检查剪贴板内容是否仍然正确

### 9. 手机输入法遮挡提交按钮
- 手机输入框获得焦点
- 让系统输入法 / 语音输入面板弹起
- 检查“送到纸杯电台”按钮是否会跟着上移
- 不收起输入法时，直接点击发送按钮是否正常
